Assignment Instructions

Mandatory Requirements

Core Gameplay Mechanics

Your game must be fully playable and implement a clear ruleset or system. If the rules aren’t common knowledge, be ready to provide your professor with reference material for the rules. Gameplay constraints like turn order/timing, collision, timing, legal plays, etc. need to be enforced. The player should be able to win or lose.

Computer-Controlled Opponent(s) or Entities

Depending on genre, this may include turn‑based decision‑making (cards, board games), pathing, movement, or attack selection (tower defense, wave‑based games), enemy spawning and behaviour logic (anything besides cards), NPC interactions (RPGs). Some form of basic, rules‑based AI is required.

Graphical User Interface

Your project must include a GUI. It should generally be clean and functional for an average person to use. It should be accessible (with mindfulness to contrast, readable text, and keyboard friendliness) and appropriate to the type of game.

Object‑Oriented Design

Your solution must somehow demonstrate the four pillars of OOP: encapsulation, abstraction, inheritance and polymorphism. It will very likely also use a library and collections in some way. The code should generally be well‑organized (e.g. use of regions, folders, and careful naming conventions).

Internal Documentation

The code body should have comment headers on all human‑generated code files/classes and meaningful comments throughout. Documentation structures that lend themselves to better development and organization, such as docstrings in Python, Javadoc in Java, or XML. Comments in C# are strongly recommended. While a style guide is not prescribed in this course, the group should agree on a style guide and adhere to it.

User Guide

Your finished project should include some form of user guide or tutorial. This can take many forms including a separate document, a short video, or thorough inline help in the game. This should include how to play, a guide to the controls, and an explanation of any settings/options.

Project Management Using GitHub

GitHub – or an approved alternative – needs to be used for source control and collaboration. As well, a GitHub Project board or another Kanban tool is essential and should be referred to and maintained from at least Week 4 onward

Optional Features (Select at Least Two)

As noted previously, not every optional feature will make sense for every genre of game so choose options that are appropriate.

Advanced Computer AI

Create a more advanced computer AI that would make alternate choices rather than the default strategy; something beyond strict rule adherence so that the opponents are more responsive, and more difficult to predict and defeat

Difficulty Scaling

Either add distinct difficulty settings, or make it so that the game becomes harder (or easier) based on time, player actions, or progress.

Opponent Variety

Create planned variation in the opponents the player can encounter. Perhaps they appear and behave differently. Note that this is not a suitable optional rule to select for an RPG since having a single enemy type would be silly.

Alternate Rulesets / Modes

Make selectable variant rules to a well‑defined game, such as alternate cards (e.g. jokers in play, wildcards) or pieces, or different board layouts. Some game styles may lend themselves well to an endless mode, or a puzzle mode.

Theming / Skins

Allow several player‑selectable UI or visual themes.
